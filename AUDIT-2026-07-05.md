## 0. Ringkasan Eksekutif (bottom line)

Dashboard **jalan dan sehat secara sintaks** (`node --check` lolos semua, Node v24.15.0, zero-dependency utuh, tidak ada secret bocor ke git). Tapi ada **3 kelas masalah yang benar-benar penting**, dan semuanya bukan soal fitur — soal **fondasi**:

1. **Keamanan control-plane** — server bind ke semua interface (`0.0.0.0`) sementara auth *opsional* dan token dikirim via query string di HTTP polos. Satu karakter `#` di `.env` = kontrol proses & tulis-file terbuka untuk seisi LAN.
2. **Keandalan proses (crash-bomb)** — beberapa jalur async & interval 25 detik memanggil `JSON.parse`/`fs.writeFileSync` **tanpa try-catch**. Config rusak sesaat saat diedit → **seluruh dashboard mati** + gateway jadi yatim (orphan).
3. **Pipa data mati di sisi produsen** — telemetry punya protokol + pembaca yang bekerja, tapi **nol penulis** (0 file `.jsonl`). Reports/ kosong, Inbox kosong, open-tasks 0, 2 dari 5 agent = entri mati. Dashboard polling agresif (6/5/25 dtk) untuk membagikan data yang sebagian besar **tidak ada**.

**Constraint #1 sistem (Theory of Constraints): produksi telemetry lintas-agent**, bukan CPU. Setelah itu constraint pindah ke **hot-path baca sinkron** (rescan vault penuh + scan 164 MB transcript tiap 5 dtk).

### Verified OK (bersih — bukan masalah)
- **Sintaks:** `node --check` lolos ketiga file JS (Node v24.15.0).
- **Git/secret:** `.env` untracked, nol history; `.gitignore` benar (`.env`, `telemetry/*.jsonl`, `avatars/*`). Repo = `main`, sinkron `origin/main` (github.com/aabrur/agentic-os, PRIVATE).
- **XSS:** frontend konsisten `esc()`; interpolasi tak-escape semua dari sumber tepercaya.
- **Command injection API:** regex rute whitelist id + action; tak ada string user ke shell.
- **Validasi avatar:** regex `data:image/(png|jpeg|webp)`, cap 3 MB decoded / 5 MB body, cek id.
- **Config↔disk:** semua path referensi ada; `claude-gateway`/`openclaw`/`copilot` resolvable di PATH.
- **Dependencies:** zero-dependency konsisten (tak ada `dependencies`/lockfile/`node_modules`) — satu-satunya gap: field `engines` kosong.

### 3.2 Theory of Constraints — 5 Focusing Steps

1. **Identify** — Constraint #1 = **produksi telemetry lintas-agent**, bukan compute. Kanal telemetry punya protokol + pembaca yang bekerja tapi **nol penulis**; Reports/Inbox/open-tasks kosong; matching Daily mati; 2/5 agent inert. Dashboard polling agresif membagikan data yang tak ada.
2. **Exploit** — Aktifkan append satu-baris yang **sudah didokumentasikan** di `telemetry/README.md` sebagai kebiasaan/hook akhir-task untuk Hermes & OpenClaw; pembaca claude-code membuktikan konsumen bekerja saat data ada.
3. **Subordinate** — Stop menyetel UI/polling (cadence 5–6 dtk sia-sia melawan feed kosong → jadikan mtime/event-driven); tunda entri mati (zcode, copilot) & gap kosmetik (avatar) sampai 2 agent live benar-benar emit telemetry.
4. **Elevate** — Bangun produsen otomatis: hook post-task telemetry di tiap agent (junction gudang skill bikin 1 implementasi terlihat semua agent), dan/atau derivasi server-side aktivitas Hermes/OpenClaw dari sumber yang sudah berdata (mtime vault, log gateway sendiri) — analog `claudeActivity()`.
5. **Repeat / pindah** — Setelah telemetry mengalir, constraint pindah ke **hot-path baca sinkron**: rescan vault penuh per-tick, scan 164.7 MB/391 transcript ulang tiap 5 dtk, dan pertumbuhan `.jsonl` tak dirotasi — lalu ke desain **single-process** (dashboard + poller + supervisor berbagi satu event loop).

### 3.3 Bottleneck runtime/operasional (ranked)
1. **`claudeActivity()` hot-path** — `server.js:303-361`: tiap 5 dtk readdir 34 dir + stat 391 `.jsonl` (164.7 MB) + tail-read 400 KB × 8 file + parse baris-per-baris, **sinkron**. Biaya naik monoton (tanpa rotasi).
2. **Rescan vault penuh per `/api/state`** — `server.js:82` tiap 6 dtk/tab + `agentVaultStatus` ×5 + `openTasks()` baca ulang. Semua sinkron; N viewer = kelipatan linear. `agentDetail` ulang walk sendiri tiap 5 dtk.
3. **Status-poll pileup** — `pollAllStatus` 25 dtk tanpa in-flight guard; tiap `gwCtl` timeout 45 dtk. Gateway lambat (hermes = exe venv Python, cold start) → shell overlap menumpuk.
4. **`detectRunning()` free-text** — `server.js:121-126`: regex bilingual atas output CLI arbitrer, default `false` → status "stopped" palsu; onboarding agent = kontrak-teks rapuh.
5. **Produksi telemetry & data fully manual** — README minta tiap agent append JSONL; adopsi nol. Onboarding agent baru butuh: bin gateway (semantik teks start/stop/status) + entri config + lane Brains + avatar manual → 2/5 mati; checklist setup doc 0/7.
6. **Log unbounded/uncaptured** — log owned-run = ring buffer RAM (800), hilang saat restart; log service native (`gateway.out.log`) tak pernah dibaca; transcript & telemetry `.jsonl` tanpa rotasi.
7. **Minor** — `loadConfig()` re-parse sinkron tiap request & beberapa kali per aksi; `CLAUDE_PROJECTS` hardcoded (`server.js:29`); cache graph cuma TTL 60 dtk, tak invalidasi saat vault berubah.

## 4. Rekomendasi (roadmap, tanpa eksekusi di fase ini)

Diurut Pareto — kerjakan **Sprint-0** dulu (≤1 hari, ~tutup 80% risiko).

**Sprint-0 (quick win, keamanan+reliability):**
1. Bind `127.0.0.1` default; wajibkan token saat bind non-loopback (tutup **S-1**).
2. Bungkus `loadConfig()` + semua callback async fs (`saveAvatar`, `readBody`) dengan try/catch; tambah handler `uncaughtException`/`unhandledRejection` (tutup **R-1, R-2**).
3. Hapus token dari query string; pakai header saja + `crypto.timingSafeEqual` (tutup **S-2**).
4. Tambah handler SIGINT/SIGTERM yang bunuh owned proc secara sinkron (tutup **R-3**).

**Sprint-1 (data pipeline — constraint #1):**
5. Seed + otomatiskan telemetry untuk Hermes & OpenClaw (hook akhir-task append JSONL) (tutup **D-1**).
6. Perbaiki heuristik daily-note & `detectRunning` (pakai exit code, bukan regex) (tutup **D-5, D-6**).

**Sprint-2 (perf):**
7. Cache `buildState`/walk dengan invalidasi `fs.watch` (mtime-keyed); offload parse transcript berat ke worker (tutup **P-1, P-2**).
8. Guard cycle junction di `walk()`; rotasi/rotasikan transcript & telemetry (tutup **R-5**).

**Sprint-3 (kebersihan):**
9. Sinkron `agents.config.json` dengan setup doc (Copilot, avatar default); `server.on("error")`; fix laporan `gwRunning`; nama file laporan resolusi detik.
